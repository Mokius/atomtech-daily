# 💻 GitHub Releases CodeQL 2.25.3 with Swift 6.3 Support and Five New C/C++ Queries

> **💻 Dev Tools** · 🟢 Notable · 2026-05-11

---

## 📋 Summary
GitHub released CodeQL 2.25.3, adding Swift 6.3 language support, five new default C/C++ security queries, and accuracy improvements across Python, Java/Kotlin, JavaScript, C#, and GitHub Actions analysis. CodeQL is GitHub's static analysis engine that automatically scans code repositories for security vulnerabilities during the development process. The C/C++ query additions are particularly significant for systems programming teams working on security-sensitive applications.

## 💡 Why It Matters
Static analysis that runs automatically in CI/CD pipelines catches vulnerabilities before they reach production — one of the most cost-effective security investments available to engineering teams. Swift 6.3 support extends these protections to iOS and macOS developers. The five new C/C++ queries specifically target vulnerability classes common in systems-level code, improving security coverage for critical infrastructure software.

## 🔍 What Is This?
CodeQL is a code scanning tool that reads your source code and automatically looks for common security vulnerabilities and coding errors — like having an automated security expert review every pull request. Version 2.25.3 adds support for newer versions of programming languages and more types of security issues it can detect.

## 🔭 Looking Ahead
As supply chain attacks on open-source software continue to rise, tools like CodeQL become increasingly critical. Expect GitHub to continue expanding CodeQL's language coverage and query libraries, eventually making comprehensive automated security analysis standard for all repositories on the platform.
