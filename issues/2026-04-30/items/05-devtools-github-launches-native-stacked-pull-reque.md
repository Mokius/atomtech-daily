# 💻 GitHub Launches Native Stacked Pull Requests via gh-stack CLI Extension

> **💻 Developer Tools** · 🟡 Medium · 2026-04-30

---

## 📋 Summary
GitHub launched native support for stacked pull requests through a new CLI extension called gh-stack, closing a workflow gap that third-party tools like Graphite and Aviator have filled for several years. The extension allows developers to create chains of dependent PRs, each building on the last, making large features easier to review in incremental slices. The feature integrates directly with GitHub's existing PR interface.

## 💡 Why It Matters
Stacked PRs are a preferred workflow for many senior engineers and large engineering teams, enabling incremental code review without waiting for an entire feature to be complete. Until now, GitHub lacked native support, forcing teams to use external tools or cumbersome workarounds. First-party support reduces friction and lowers the barrier to adopting this practice.

## 🔍 What Is This?
A stacked pull request is when you break a large code change into a series of smaller, sequential PRs — each one depends on the previous being merged. Instead of one enormous diff, reviewers see focused, digestible slices. GitHub's new gh-stack CLI extension makes creating and managing these chains straightforward directly from the terminal.

## 🔭 Looking Ahead
Native stacked PR support may push more engineering teams to adopt trunk-based development patterns and smaller, more frequent commits. Third-party tools in this space like Graphite will need to differentiate further or risk losing ground to GitHub's built-in solution.
