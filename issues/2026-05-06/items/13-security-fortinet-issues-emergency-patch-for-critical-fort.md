# 🔐 Fortinet Issues Emergency Patch for Critical FortiClient EMS Zero-Day CVE-2026-35616

> **🔐 Security** · 🔴 High · 2026-05-06

---

## 📋 Summary
Fortinet has disclosed CVE-2026-35616, a critical improper access control vulnerability in its FortiClient Endpoint Management Server (EMS) software with a CVSS score of 9.1. The flaw allows unauthenticated attackers to execute code through crafted HTTP requests. The vulnerability has already been exploited in the wild, and Fortinet urges all customers to apply the emergency hotfix immediately.

## 💡 Why It Matters
FortiClient EMS is deployed across thousands of enterprise environments for endpoint security management. A 9.1 CVSS unauthenticated remote code execution vulnerability in a widely-deployed security product is among the most severe categories of enterprise risk. Active exploitation means attackers are already targeting unpatched systems — rapid patching is not optional.

## 🔍 What Is This?
FortiClient EMS is security software used by companies to manage the security settings of employee computers. CVE-2026-35616 is a flaw that lets attackers take control of the EMS server without needing a username or password. It has already been used in real attacks, making it urgent for organizations to apply the fix.

## 🔭 Looking Ahead
Expect CISA to add CVE-2026-35616 to its Known Exploited Vulnerabilities catalog. Organizations that use Fortinet EMS should treat this as a P0 patching priority and audit access logs for signs of compromise before and after patching.
